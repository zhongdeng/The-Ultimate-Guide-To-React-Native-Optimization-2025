# Concurrent React

应用加载和响应用户交互的速度（以及流畅度）在用户感知中有时比实际的性能更重要——尤其是在从远程服务器获取资源时，这通常是许多应用中最慢的部分。虽然你可能无法让你的应用在物理层面上运行得更快，但你可以提升它给用户带来的“快”的感觉。由于这关系到相对的感知，我们称之为感知性能（perceived performance）。

提升感知性能的一个实用准则是：相比让用户等待操作完全完成，通常更好地做法是快速给予响应并定期更新状态。这也是 Concurrent React 的核心理念之一。

Concurrent React 是对 React 渲染模型的一次基础性更新，它在 React 18 中引入，并在 React Native 的 0.69 版本中实现。与一次性完成更新的传统方式不同，Concurrent React 允许更新被暂停、重新排序，甚至在必要时放弃。这确保了诸如用户输入这样的关键交互可以作为最高优先级被处理，从而提升整体用户体验。

虽然并发性是一个幕后的机制，但它解锁了多个强大的功能，包括可中断渲染（interruptible rendering）、时间分片（time-slicing）和自动批处理（automatic batching）。这些能力使 React Native 能够更高效地准备和管理 UI 更新，特别是在渲染大型数据集或处理复杂交互等资源密集型场景中。

> 要在 React Native 中使用 Concurrent React 的特性，比如 `useTransition`、`useDeferredValue`、自动批处理以及用于数据获取的 `Suspense`，你必须迁移到 新架构。从 React Native 0.76 起，新项目默认使用新架构。

## 处理渲染缓慢的组件

在许多应用中，某些组件渲染成本高且难以优化——例如，当执行复杂计算或集成第三方库时。你可以使用 `useDeferredValue` 钩子来防止这些繁重组件干扰更高优先级的任务，而不是让它们阻塞你的 UI。

`useDeferredValue` 钩子的工作方式就像是你应用更新的一个缓冲区。想象你正在写一张便条，而你的朋友正在你肩膀后面阅读。如果他打断你并评论你写的内容，你可能需要暂时停下。类似地，`useDeferredValue` 允许 React“暂停”那些不太关键的更新，以确保诸如打字或点击之类的用户交互保持流畅。

在实践中，它确保依赖于频繁变化值的组件不会拖慢你的应用。相反，React 会优先渲染当前最重要的 UI 部分，例如在用户打字时更新显示，而将更重的计算或更新延后到后台处理。

```tsx
import { ActivityIndicator, Button } from "react-native";
import { useDeferredValue, useState } from "react";
import CounterNumber from "@/components/CounterNumber";
import SlowComponent from "@/components/SlowComponent";

function DeferredScreen() {
  const [count, setCount] = useState(0);
  const deferredCount = useDeferredValue(count);

  return (
    <>
      <CounterNumber count={count} />
      <SlowComponent count={deferredCount} />
      {count !== deferredCount ? <ActivityIndicator /> : null}
      <Button onPress={() => setCount(count + 1)} title="Increment" />
    </>
  );
}

export default DeferredScreen;
```

在上面的示例中，`useDeferredValue` 钩子确保了对 **SlowComponent** 的更新会被延后，而 **CounterNumber** 的响应性仍保持良好。每当延迟值落后于即时值时，**ActivityIndicator** 就会出现，为用户提供清晰的视觉反馈。这种模式在渲染代价高昂的组件或执行耗时计算时特别有用。

> 请记住，把传递了延迟值的计算密集型组件用 React.memo() 包裹起来，以防止由于父组件重新渲染而导致的不必要重新渲染。

### 在等待更新时展示旧值

在处理异步操作（如数据获取）时常见的做法是(在每个应用中都会遇到)，在操作完成前显示一个加载状态。但这可能会导致一种生硬的用户体验，内容频繁消失并被加载指示器取代。

`useDeferredValue` 提供了一种更为精细的方式，它允许你在更新尚未完成时展示旧值。

```tsx
import { useState, useDeferredValue, Suspense } from "react";
import { View, TextInput } from "react-native";
import SearchResults from "./SearchResults";

function SearchScreen() {
  const [query, setQuery] = useState("");
  const deferredQuery = useDeferredValue(query);

  return (
    <View>
      <TextInput
        value={query}
        onChangeText={setQuery}
        placeholder="Search items..."
      />
      <Suspense fallback={<LoadingSpinner />}>
        <SearchResults query={deferredQuery} />
      </Suspense>
    </View>
  );
}
```

在初始渲染期间，`query` 和 `deferredQuery` 值是相同的。当用户在 TextInput 中开始输入时，`query` 属性会立刻更新，但 `deferredQuery` 会维持旧值，直到更新（搜索操作）完成。

你可以通过引入一个属性，在搜索结果仍为旧数据时保留旧值，从而改善视觉反馈并更好地呈现这种体验：

```diff
import { useState, useDeferredValue, Suspense } from 'react';
import { View, TextInput } from 'react-native';
import SearchResults from './SearchResults';

function SearchScreen() {
  const [query, setQuery] = useState('');
  const deferredQuery = useDeferredValue(query);
+ const isStale = query !== deferredQuery;

  return (
    <View>
      <TextInput
        value={query}
        onChangeText={setQuery}
        placeholder="Search items..."
      />
      <Suspense fallback={<LoadingSpinner />}>
-       <SearchResults query={deferredQuery} />
+       <View style={[
+         isStale && { opacity: 0.8 }
+       ]}>
+         <SearchResults query={deferredQuery} />
+       </View>
      </Suspense>
    </View>
  );
}
```

当我们等待更新时，搜索结果的透明度会降低。一旦更新完成，新结果将被呈现，同时 `deferredQuery` 将等于 `query`。

## 使用 `过渡` 来处理非关键更新

`过渡`允许开发者处理非关键的 UI 更新，例如次要状态的更改，方法是将它们标记为低优先级。这确保了诸如用户输入这样的紧急交互优先被处理。与 `useDeferredValue` 延迟的是某个具体值的渲染不同，`useTransition` 延迟的是整个更新过程，从而更平滑地管理状态变更。

```tsx
import { Button, SafeAreaView, StyleSheet } from "react-native";
import { useState, useTransition } from "react";
import CounterNumber from "@/components/CounterNumber";
import SlowComponent from "@/components/SlowComponent";
function TransitionScreen() {
  const [count, setCount] = useState(0);
  const [slowCount, setSlowCount] = useState(0);
  const [isPending, startTransition] = useTransition();
  const handleIncrement = () => {
    setCount((prevCount) => prevCount + 1);
    startTransition(() => {
      setSlowCount((prevSlowCount) => prevSlowCount + 1);
    });
  };

  return (
    <SafeAreaView style={styles.container}>
      <CounterNumber count={count} />
      <SlowComponent count={slowCount} />
      {isPending ? <LoadingSpinner /> : null}
      <Button onPress={handleIncrement} title="Increment" />
    </SafeAreaView>
  );
}
export default TransitionScreen;
```

通过将关键与非关键更新区分开来，UI 保持响应性并提供流畅的用户体验。`useTransition` 钩子处理如 `slowCount` 状态这样的非紧急更新，确保像 **CounterNumber** 中的 `count` 状态这样的关键更新被即时处理，而非关键更新则在后台进行。

## 如何在 `useDeferredValue` 和 `useTransition` 之间选择

`useDeferredValue` 最适用于延迟单个值的渲染。这确保了依赖该值的组件不会阻塞关键更新。例如，当你根据频繁变化的输入渲染一个计算密集型组件时，`useDeferredValue` 能确保应用的其余部分依然保持响应。

而 `useTransition` 则是通过将整个更新或渲染任务标记为低优先级来延迟执行。当你需要在后台更新多个状态或组件时，它最为有效，例如在视图之间过渡或更新 UI 的更大部分时。在 React Native 应用中，视图切换通常由某种原生导航控制，比如带有原生堆栈导航器的 React Navigation。但某些导航器是基于 JavaScript 的，因此可以适用于此优化技术。

要在两者之间做出选择，可以根据更新的范围来判断。如果你在管理单个属性或值，那么 `useDeferredValue` 更适合；如果你在处理涉及多个更新或更广泛 UI 变化的过渡，那么 `useTransition` 更合适。

## Automatic Batching

React 18 引入了自动批处理，即将多个状态更新合并为一次重渲染的过程，以提升性能。在此之前，React 只会在 React 的事件处理器内部进行批处理。对于 Promise、setTimeout、原生事件处理器或其他任何位置的更新，React 默认并不会进行批处理。

```js
// Before: only React events were batched.
setTimeout(() => {
  setProductQuantity((quantity) => quantity + 1);
  setIsCartOpen((isOpen) => !isOpen);
  // React will render twice, once for each state update (no batching)
}, 1000);
// After: updates inside of timeouts, promises etc.
setTimeout(() => {
  setProductQuantity((quantity) => quantity + 1);
  setIsCartOpen((isOpen) => !isOpen);
  // React will only re-render once at the end (thanks to batching!)
}, 1000);
```

自动批处理还能防止你的组件处于“半完成”状态，即只有一个状态更新，这可能会导致意外的视觉错误。

> 如果你想更深入了解[这个](https://react.dev/learn/queueing-a-series-of-state-updates)主题，我们强烈推荐阅读 React 官方文档中的这篇文章。

**Concurrent React** 的功能，包括 `useDeferredValue`、`useTransition` 和自动批处理，能通过智能管理渲染任务和状态更新而不阻塞 UI，从而提升感知性能。这些强大的能力可以在你的应用中逐步采用，帮助你在资源密集型场景下增强性能和响应性，尤其是在最需要的地方。
