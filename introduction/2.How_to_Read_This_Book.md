# 如何阅读本书

这本书适合各种经验水平的 React Native 开发者阅读。我们相信，无论你是刚接触 React Native 的新手，还是有丰富经验的老手，不管你之前是做 Web 开发还是原生应用开发，都能从中找到对你有价值、能应用在你项目中的内容。

当然，我们也理解，在某些时候你可能对书中提到的所有优化技巧都不感兴趣。比如有时候你可能只专注于优化 React 的重复渲染（re-render）问题。所以，虽然你完全可以顺着章节顺序从头读到尾，我们也特别设计了每一章都可以单独阅读，你可以直接跳到你感兴趣的主题上手。

## 本书包含哪些内容？

为了让大家更容易找到自己关心的内容，我们把这本指南分成了三个部分，分别聚焦于不同类型的优化方向：
**React 层面的优化**、**原生（Native）层面的优化**，以及 **整体构建时（Build-Time）** 的优化。每个部分都包括：
**主题导读**、**操作指南**、**最佳实践**。
它们将帮助你提升应用最核心的性能指标：
**FPS（帧率，Frames Per Second）** 和 **TTI（可交互时间，Time to Interactive）**。

下面是每个章节中你可以期待的内容：

- **导读**：介绍与该部分相关的核心概念、术语和思路
- **操作指南**：讲解如何使用专门的工具，以及如何测量关键性能指标
- **最佳实践**：提供具体建议，帮你让应用启动更快、运行更流畅

## 本书使用的说明方式

为了让内容更清晰，我们在书中加入了很多简洁的代码片段、工具截图以及示意图。

我们也会在合适的位置链接外部资源，比如官方文档、库，以及本书的其他章节内容。

另外，对于某些概念，我们还专门做了补充说明（Callout 小贴士形式）。这些内容虽然不是线性阅读的一部分，但我们强烈建议你不要跳过，它们能帮助你更深入理解上下文。
