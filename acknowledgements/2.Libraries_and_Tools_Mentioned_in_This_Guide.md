# 本书提到的库和工具

### 动画与性能优化工具

- [React Native Reanimated](https://docs.swmansion.com/react-native-reanimated/)：高性能的 React Native 动画库。

- [React Navigation](https://reactnavigation.org/)：支持动画过渡的 React Native 导航库。

### React 编译器与状态管理

- [Jotai](https://github.com/pmndrs/jotai)：极简的原子状态管理库。

- [Zustand](https://github.com/pmndrs/zustand)：可扩展的状态管理库，比 Redux 更简单。

- [Recoil](https://recoiljs.org/)：Meta 出品的状态管理库，具备细粒度响应式特性。

### 列表与滚动性能

- [FlashList](https://shopify.github.io/flash-list/docs/)：Shopify 开发的高性能 FlatList 替代方案。

- [LegendList](https://github.com/LegendApp/legend-list)：面向 React Native 新架构优化的 FlatList 替代方案。

- [RecyclerListView](https://github.com/Flipkart/recyclerlistview)：高性能列表库，FlashList 内部也使用了该机制。

### 性能分析与调试工具

- [React Native DevTools](https://reactnative.dev/docs/react-native-devtools)：内置的调试与性能分析工具。

- [Chrome DevTools](https://developer.chrome.com/docs/devtools)：浏览器中的调试工具，可用于内存与性能分析。

- [Flashlight](https://flashlight.dev/)：可在 Android 上测量 JS FPS 和收集性能指标。

- [Lighthouse](https://developer.chrome.com/docs/lighthouse/overview/)：Google 出品的网页性能审计工具，常用于基准参考。

- [Xcode Instruments](https://help.apple.com/instruments/mac/current/)：macOS 下用于内存、CPU 和性能瓶颈分析的工具。

- [Android Studio Profiler](https://developer.android.com/studio/profile)：Android 应用 CPU、内存、网络和电池使用情况分析工具。

- [Perfetto](https://ui.perfetto.dev/)：用于深入分析 Android 应用系统调用和性能的追踪工具。

### JavaScript 引擎与原生技巧

- [Hermes](https://github.com/facebook/hermes/blob/main/README.md)：专为 React Native 优化的轻量高性能 JS 引擎。

- [Yoga](https://www.yogalayout.dev/)：React Native 使用的跨平台布局引擎。

### 代码优化与压缩

- [R8](https://developer.android.com/build/shrink-code)：Android 默认的 APK 压缩、优化与混淆工具，替代 ProGuard。

### 代码分割与远程加载

- [Re.Pack](https://re-pack.dev/)：基于 Webpack 的 React Native 打包工具，支持代码分割与远程模块加载。

- [Module Federation](https://webpack.js.org/concepts/module-federation/)：Webpack 功能，支持微前端架构与动态加载。

- [Zephyr Cloud](https://zephyr-cloud.io/)：支持模块联邦与微前端部署管理的平台。

### Tree Shaking

- [Rspack](https://rspack.dev/)：类 Webpack 的高性能打包器，支持 React Native 的 tree shaking。

- [Terser](https://terser.org/)：JS 混淆压缩工具，用于减小包体积并提升运行性能。

- [metro-serializer-esbuild](https://microsoft.github.io/rnx-kit/docs/tools/metro-serializer-esbuild)：集成 Esbuild 的 Metro 插件，支持 tree shaking。

### JS 与 App 包体积分析

- [source-map-explorer](https://www.npmjs.com/package/source-map-explorer)：分析 JS 包内容并识别未使用依赖。

- [Expo Atlas](https://github.com/expo/atlas#readme)：用于 Expo 项目的 JS 包可视化分析工具。

- [webpack-bundle-analyzer](https://www.npmjs.com/package/webpack-bundle-analyzer)：Webpack 插件，提供交互式 bundle 大小可视化。

- [bundle-stats](https://github.com/relative-ci/bundle-stats)：CLI 工具，生成 JS bundle 分析报告。

- [Statoscope](https://statoscope.tech/)：支持 Webpack 与 Rspack 的深入 bundle 分析工具。

- [Rsdoctor](https://rsdoctor.dev/)：用于分析 Rspack 构建的 React Native bundle。

- [Emerge Tools](https://www.emergetools.com/)（付费）：Android/iOS 应用体积分析与优化平台。

- [Ruler](https://github.com/spotify/ruler)：由 Spotify 开发的 Gradle 插件，用于分析 APK 和 AAB 大小。

### 第三方库体积分析

- [Bundlephobia](https://bundlephobia.com/)：查看 npm 包最小化与压缩后的大小。

- [pkg-size.dev](https://pkg-size.dev/)：类似于 Bundlephobia 的 Web 工具。

- [Import Cost](https://marketplace.visualstudio.com/items?itemName=wix.vscode-import-cost)（VS Code 插件）：VS Code 扩展，实时显示引入模块的体积。

### 原生开发工具与构建系统

- [React Native Builder Bob](https://github.com/callstack/react-native-builder-bob)：用于搭建支持新架构的 React Native 库的脚手架工具。

- [Ninja](https://ninja-build.org/)：React Native 项目中常用于编译原生代码的高效构建系统。

### 交互时间指标分析（TTI）

- [react-native-performance](https://github.com/oblador/react-native-performance)：追踪 React Native 应用 TTI 等性能指标的工具。
