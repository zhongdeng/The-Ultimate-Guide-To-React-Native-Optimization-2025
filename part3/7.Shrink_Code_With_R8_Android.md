# 在 Android 中使用 R8 压缩代码

如[《如何分析 App 安装包的大小》](./2.How_to_Analyze_App_Bundle_Size.md)一章所述，应用体积是一个非常重要的指标，你应当时刻关注。在前几章中，我们讨论了如何优化 JavaScript Bundle 的体积。但在某些情况下，原生代码也会影响应用的总体积。本节介绍的一些工具可以帮助你进一步减少应用的体积，同时还可以通过代码混淆提升安全性。

## 启用 R8

你可能听说过 ProGuard。这是 Android Gradle 中的一个工具，用于压缩、优化并混淆你的应用代码，从而减小 APK 的体积。然而，如果你的项目使用的是 React Native v0.60 或更高版本，那么 Android Gradle 插件（大多数 Android 项目的核心依赖）就不再使用 ProGuard 来优化代码了，而是使用了 R8。

R8 是 Android 上用于压缩、优化和混淆 APK 的新一代工具。它不仅性能更好，还支持现代特性（如 Kotlin），这是 ProGuard 所不具备的。同时，R8 也更快，并兼容 ProGuard 的配置文件（ProGuard rules），因此你在 Android 项目中仍然会看到一些 “proguard” 的引用，这是正常的。

要在 React Native 项目中启用 R8，请编辑 **app/build.gradle** 文件：

```gradle
def enableProguardInReleaseBuilds = true
```

在默认的 React Native 模板中，这个变量会将 `minifyEnabled` 设置为 true，作用于 release 构建变体，这就告诉 Gradle 在打包分发版本时应用 R8 的优化。

在[《如何分析 App 安装包的大小》](./2.How_to_Analyze_App_Bundle_Size.md)一章中，我们测试了使用默认的 ProGuard 规则启用 R8 后对应用体积的影响。示例应用在未优化的情况下体积为 9.5 MB，启用 R8 后缩减为 6.3 MB，减少了 33%。对于更大的项目，虽然提升比例可能略低，但仍然是值得尝试的优化方式。

### R8 的 ProGuard 规则和配置

如前所述，R8 使用 “ProGuard 规则” 进行配置。在一个专门的规则文件中，你可以控制哪些类、方法和字段在构建优化过程中需要保留（preserved）或可以被移除（discard）。你还可以配置代码混淆的规则（比如类名和方法名的重命名），以增加代码被反编译的难度。在 React Native 应用中，这些规则定义在 **app/android/proguard-rules.pro** 文件中。该文件默认是空的。

在大多数情况下，你无需手动添加规则。但如果在启用死代码删除（dead code elimination）后，应用出现运行异常或功能缺失的情况，你可能需要手动保留某些类。这种情况通常出现在某些依赖库假设类名不会变化时。例如，如果你使用 Firebase 并遇到相关问题，可以告诉 R8 保留这些类，避免其被移除：

```proguard
-keep class com.google.firebase.** { *; }
```

当你设置了 `minifyEnabled`，混淆默认就已经开启了。如果你出于某些原因希望禁用混淆，可以在 **proguard-rules.pro** 文件中添加如下设置：

```proguard
-dontobfuscate
```

你可以阅读 ProGuard [官方文档](https://developer.android.com/build/shrink-code) 了解更多规则定义方式。

### 压缩资源（Shrinking Resources）

当你在 release 构建中启用了 R8，还可以进一步开启 Gradle 的资源压缩功能（resource minification），以进一步减小包体积。启用后，Gradle 将执行以下优化操作：

1. 合并重复资源。
2. 优化 PNG 图像（降低颜色深度并应用无损压缩）。
3. 处理矢量图（vector drawable）文件。

你可以在 **app/build.gradle** 中通过 **shrinkResources** 设置启用该功能：

```gradle
android {
  buildTypes {
    release {
      // Enables code shrinking, obfuscation, and optimization for only your project's release build type. Make sure to use a build variant with `debuggable false`.
      minifyEnabled true

      // Enables resource shrinking, which is performed by the Android Gradle plugin.
      // Make sure `minifyEnabled` is set to `true`
      shrinkResources true
    }
  }
}
```

现在你已经了解了如何通过启用 死代码删除（dead code elimination） 来减小应用体积。记得在应用启用该技术后发布到商店之前，一定要进行充分的测试。你也可以使用 Ruler 工具 来对比应用的下载体积和安装体积差异，以验证优化效果。
